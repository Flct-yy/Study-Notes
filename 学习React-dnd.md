# 学习React-DnD

## 理解基本概念

**拖动源（Drag Source）** 是给组件添加“拖拽”的能力。比如给待办卡片组件加了这个，鼠标点住卡片就能拖动了。

**拖放目标（Drop Target）** 是给组件添加接受“拖放”的能力。比如给 “已完成” 列表组件加了这个，就能把待办卡片拖进去。

**项目（Item）** 是 React-DnD 里用来 “描述被拖动东西” 的普通 JS 对象，它包含当前拖拽内容的核心属性，可以为你在不同 拖拽源 和 放置目标 中传递关键信息。比如拖待办卡片时，项目可以是 ```{ todoId: 1, title: '学习Javascript' }```，不用带组件所有数据，只存传递需要的核心信息

**类型（Type）** 是用来给 项目（Item） 分类的标识符（通常是字符串或Symbol），当 拖拽源 和 放置目标 过多的时候，用来区分 “哪些拖拽源和放置目标可以交互” 的。比如给所有待办卡片定类型 'TODO'，给所有待办列表的拖放目标也关联类型 'TODO'，这样只有 “TODO” 类型的拖动源能拖进 “TODO” 类型的目标，不会出现把其他类型的拖进待办列表的混乱情况。

**监视器（Monitor）** 是用来 监听并获取 拖拽状态的工具，核心就是 向外暴露当前拖拽状态，当拖拽状态改变时，可以根据拖放状态的变化更新组件的样式。比如能知道“现在有没有东西在拖拽”、“当前组件有没有被拖拽的组件悬停着”、“这个东西能不能放到当前的组件上”，这样组件就能靠这些状态调整 UI（比如悬停时变背景色、可放置时加边框）。

**连接器（Connector）** 是用来将 React 组件的 DOM 节点与拖放逻辑 “绑定” 的工具。用来告诉React-DnD“这个DOM节点是可拖拽源”或“那个DOM节点是拖放目标”，让底层能处理其拖拽事件。

**拖放后端** 将原生的DOM事件（比如鼠标按下、移动...）转化为统一识别的内部事件。初学只要知道“选择一个能用的后端（例如PC端默认选择HTML5后端），组件就能实现正常拖放”就行。
